## 2024-05-18 - Caching Large Extension Artifacts in Android WebViews
**Learning:** Network interceptors (`shouldInterceptRequest`) run on a critical background thread for every single asset requested by a web page. In this codebase, Chrome extensions are stored as Base64 encoded Zip archives in `SharedPreferences` (`pluginsJson`). Parsing multi-megabyte JSON arrays, decoding Base64 strings, and extracting files from Zip streams inside network interceptors for every asset request caused a severe bottleneck.
**Action:** Always employ an `LruCache` for extracted byte streams (`ByteArray`) mapping to file paths within the Zip. Additionally, memoize the parsed data structures (`JSONArray` mapped to list objects) relying on simple checksums/hashes (like `String.hashCode()`) to invalidate the memory cache instead of reparsing the `SharedPreferences` JSON for every plugin-related operation.
