## $(date +%Y-%m-%d) - Cached Plugin JSON Parsing and Extension ZIP Decoding
**Learning:** In the ConsoleFlow application, plugins were being read from `SharedPreferences` and parsed as JSON arrays on every single page load and every intercepted resource request. Furthermore, Chrome extensions stored as Base64 encoded strings in `SharedPreferences` were being decoded into byte arrays multiple times per request. This created a massive memory overhead and computation bottleneck for `shouldInterceptRequest`.
**Action:** Always verify how large objects (like Base64 encoded files or heavy JSON representations) are fetched in Android WebViews and intercepts. Cache them in-memory (e.g., using `ConcurrentHashMap` for multithreaded requests) and invalidate the cache only on updates (e.g., `upsertPlugin`, `removePlugin`).
